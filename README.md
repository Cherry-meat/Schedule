# StudentPlanner 

**StudentPlanner** — это система управления персональным расписанием студента, созданное для того, чтобы помочь студентам эффективно управлять своим учебным процессом. Оно объединяет расписание занятий, список задач и статистику в одном удобном интерфейсе, доступном из любого браузера.

## О проекте

### Почему был создан этот продукт?

Студенческая жизнь наполнена множеством задач: лекции, семинары, дедлайны курсовых и лабораторных работ. Держать всё это в голове или в разрозненных блокнотах и таблицах — сложно и неэффективно. Этот проект родился из практической необходимости создать единый центр управления учебой, который показывает в себе простоту планирования.

### Какую проблему он решает?

Приложение решает проблему организации учебного времени, позволяя студентам никогда не пропускать пары благодаря структурированному расписанию, вовремя сдавать все работы через отслеживание задач и приоритетов, а также анализировать свою нагрузку и прогресс с помощью встроенной статистики.

### В чём его ключевые преимущества?

*   **Специализация под учебу:** Интерфейс и логика приложения заточены именно под образовательный процесс.
*   **Полный цикл управления:** От создания события или задачи до отслеживания его выполнения и анализа статистики.
*   **Наглядный интерфейс:** Чистый, современный дизайн с удобной навигацией и визуальными подсказками.

## Технологии

Проект построен на классическом стеке веб-технологий и состоит из двух основных частей: клиентской (Frontend) и серверной (Backend).

**Клиентская часть (Frontend):**
*   **Язык:** JavaScript (ES6+)
*   **Библиотека:** React.js для построения пользовательского интерфейса.
*   **Маршрутизация:** React Router v6 для навигации между страницами.
*   **HTTP-клиент:** Axios для взаимодействия с сервером.
*   **Стилизация:** Чистый CSS (файл `App.css`) с использованием CSS-переменных для кастомизации.

**Серверная часть (Backend):**
*   **Язык:** Go (Golang) — высокопроизводительный и надежный язык.
*   **База данных:** PostgreSQL — мощная объектно-реляционная база данных.
*   **Маршрутизация:** Gorilla Mux — гибкий роутер для Go.
*   **Middleware:** Gorilla Handlers (логирование), самописный CORS-обработчик.
*   **Безопасность:** Bcrypt для хеширования паролей.

## Быстрая установка и запуск

Вы можете запустить приложение локально, выполнив следующие шаги:

### 1. Предварительные требования
Убедитесь, что на вашем компьютере установлены:
*   **Node.js** (версии 16 или выше) и **npm**: [Скачать](https://nodejs.org/)
*   **Go** (версии 1.21 или выше): [Скачать](https://go.dev/dl/)
*   **PostgreSQL** (версии 12 или выше): [Скачать](https://www.postgresql.org/download/)

### 2. Настройка базы данных (PostgreSQL)
1.  Запустите сервис PostgreSQL.
2.  Создайте базу данных для проекта:
    ```sql
    CREATE DATABASE student_planner;
    ```
3.  Убедитесь, что у вас есть пользователь (например, `postgres`) с паролем (по умолчанию `admin`), или измените параметры подключения в файле `server/database.go`.

### 3. Запуск сервера (Backend)
1.  Откройте терминал и перейдите в директорию `server`.
2.  Установите необходимые Go-модули (они подтянутся автоматически при сборке).
3.  Запустите сервер:
    ```bash
    go run .
    ```
    Вы должны увидеть сообщения:
    ```
    База данных подключена
    Таблицы созданы/проверены
    Сервер запущен на http://localhost:8080
    ```

### 4. Запуск клиента (Frontend)
1.  Откройте **новый** терминал и перейдите в корневую директорию проекта (где находится `package.json`).
2.  Установите зависимости:
    ```bash
    npm install
    ```
3.  Запустите React-приложение:
    ```bash
    npm start
    ```
    Приложение автоматически откроется в браузере по адресу `http://localhost:3000`.

## Возможности

StudentPlanner предоставляет студентам все необходимые инструменты для учебы:

*   **Система пользователей:** Регистрация и авторизация с безопасным хешированием паролей. Данные каждого пользователя изолированы.
*   **Управление расписанием:**
    *   Создание, редактирование и удаление событий (лекции, практики, экзамены).
    *   Указание даты, времени, продолжительности, места проведения и преподавателя/предмета.
    *   Просмотр всех событий в удобном списке.
*   **Управление задачами:**
    *   Создание задач с описанием, приоритетом (Высокий, Средний, Низкий) и сроком выполнения.
    *   Отметка о выполнении задачи.
    *   Фильтрация на "Ожидающие" и "Выполненные".
*   **Статистика обучения:**
    *   Общее количество занятий и задач.
    *   Процент выполненных задач.
    *   Суммарное количество часов, проведенных на занятиях.
    *   Наглядное распределение событий по типам (лекции/практики/экзамены).
*   **Адаптивный дизайн:** Интерфейс удобно использовать как на компьютере, так и на планшете или смартфоне.

## Использование

1.  **Начало работы:** При первом посещении вы попадете на лендинг. Зарегистрируйтесь или войдите в систему.
2.  **Главная панель (Dashboard):** После входа вы увидите приветствие, сводку на сегодня и список недавно добавленных событий.
3.  **Расписание (Schedule):** Здесь отображаются все ваши учебные события. Вы можете добавить новое, нажав на кнопку "Добавить событие" и заполнив форму.
4.  **Задачи (Tasks):** Управляйте списком дел. Добавляйте задачи, указывайте приоритет и дедлайн, отмечайте выполненные.
5.  **Статистика (Statistics):** Отслеживайте свой прогресс. Здесь собрана вся аналитика по вашей учебной активности.

## Разработка и тестирование

Проект имеет четкую структуру, упрощающую разработку и тестирование.

*   **Frontend:**
    *   `components/`: React-компоненты для каждой страницы и элемента интерфейса.
    *   `services/`: Модуль `api.js` для централизованной работы с сервером.
    *   `App.js`: Главный компонент с настройкой роутинга и состоянием авторизации.
*   **Backend:**
    *   `main.go`: Точка входа, настройка роутера и CORS.
    *   `handlers.go`: Содержит все обработчики (контроллеры) API.
    *   `database.go`: Инициализация подключения к БД и создание таблиц.
    *   `models`: Структуры данных (`User`, `Event`, `Task`).

Проведено комплексное тестирование, охватывающее основные сценарии использования:

| **Т1** | Полная регистрация и вход | Аутентификация, создание пользователя |
